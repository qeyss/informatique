Ubuntu serveur: Perte mot de passe root

Posted on 26/08/2013 by fred

Mise à jour de l’article!
voici une astuce qui permet de redéfinir le mot de passe perdu pour le compte root.

Ancienne méthode

(date du 29/06/2011)
– A l’invite de grub (début du démarrage linux) appuyer sur la touche « e »
– Sélectionner la deuxième ligne commençant par kernel /vmlinuz-2.6…. et appuyer sur la touche « e »
– remplacer « ro splash quiet » par « rw init=/bin/bash » puis appuyer sur entrer (attention le clavier est en QWERTY )
– Appuyez sur la touche b afin de booter em mode single user

A l’invite de commande vous pouvez changer le mot de passe root avec la commande « passwd »

Méthode à partir d’Ubuntu 12.04Lts

  • Au menu du Grub: choisir « Mode de dépanage »

  • Au menu de récupération, choisir: root

  • Une fois arriver au shell, saisir les commandes suivantes:

    • Remonter la partition en lecture et écriture:

    mount -rw -o remount /

    Changement du mot de passe root:

    passwd root

    En image:

    A ce stade, si le compte root n’a pas été définie il est aussi possible d’ajouter un utilisateur:

    adduser fred

    Une fois le mot de passe redéfinie, reboot de la machine:

    reboot
